## 一、项目定位

**anni-ui** 是一个轻量级的 React 中后台 UI 组件库，基于 **Design Token + Tailwind CSS** 构建，目标是为中后台项目提供统一、可控、易扩展的基础 UI 能力。

与 Ant Design 等重量级方案相比，它的核心优势是：
- 体量轻，样式通过 CSS 变量 + Tailwind 原子类管理，不依赖复杂 CSS-in-JS
- 主题切换零运行时开销（纯 CSS 变量覆盖）
- API 类型安全（TypeScript 全覆盖）
- 满足基础无障碍（a11y）要求

## 六、主题系统详解

### 工作原理

```
用户调用 setMode('dark')
  ↓
ThemeProvider 更新 state，写入 localStorage
  ↓
document.documentElement.classList.add('dark')
  ↓
.dark 规则覆盖 CSS 自定义属性
  ↓
所有引用 var(--color-*) 的组件自动更新颜色（无 React re-render，纯 CSS）
```

- `system` 模式额外监听 `MediaQueryList.change`，操作系统切换主题时实时同步
- localStorage 不可用（隐私模式）时不报错，降级为 `defaultMode`

### 颜色 Token（明亮 / 暗黑）

| Token 前缀 | 明亮值（代表色） | 暗黑值（代表色） |
|------------|-----------------|-----------------|
| `primary` | violet-600 (#7c3aed) | violet-500 (#8b5cf6) |
| `danger` | red-600 (#dc2626) | red-500 (#ef4444) |
| `success` | green-600 (#16a34a) | green-500 (#22c55e) |
| `warning` | yellow-600 (#ca8a04) | yellow-500 (#eab308) |
| `info` | blue-500 (#3b82f6) | blue-400 (#60a5fa) |
| `neutral-50`（背景）| gray-50 (#f9fafb) | zinc-900 (#18181b) |
| `neutral-900`（文字）| gray-900 (#111827) | zinc-50 (#fafafa) |

## 七、样式系统

采用 **Tailwind CSS + CSS 自定义变量** 双轨制：

1. `index.css` 定义 `:root`（明亮）和 `.dark`（暗黑覆盖）下所有语义颜色 token
2. `tailwind.config.js` 将 CSS 变量映射为 Tailwind `colors` 扩展，可直接用 `bg-primary`、`text-danger-fg` 等类名
3. `tailwind.config.js` 配置 `darkMode: 'class'`，配合 ThemeProvider 自动切换
4. 工具函数 `cn(...inputs)` = `twMerge(clsx(inputs))`，所有组件统一使用

## 八、无障碍（a11y）规范

| 措施 | 涉及组件 |
|------|----------|
| `focus-visible` 焦点环（键盘导航可见，鼠标不触发）| Button、Input 等所有可交互组件 |
| `aria-invalid` 驱动错误样式 | Input、Checkbox、Radio、Switch、Slider |
| `role="status"` + `aria-label="加载中"` | Spinner |
| `<nav aria-label="breadcrumb">` + `aria-current="page"` | Breadcrumb |
| `role="radiogroup"` | RadioGroup |
| `role="switch"` + `aria-checked` | Switch |
| `role="slider"` + `aria-valuenow/min/max` | Slider |
| FormControl 自动注入 `aria-describedby` 关联 FormMessage | Form 体系 |
| 语义化 HTML 标签（`<header>`、`<aside>`、`<main>`、`<footer>`）| Container 系列 |

## 九、目录规范

每个组件均遵循以下目录约定：

```
src/core/组件名/
├── index.ts              ← 对外导出入口
├── 组件名.tsx             ← 组件实现
├── 组件名.types.ts       ← Props 类型定义（供外部消费）
└── 组件名.test.tsx       ← 单元测试（与组件同目录）
```

全局测试基础设施统一放置在：

```
src/test/
├── setup.ts              ← jest-dom 引入 + window.matchMedia mock（由 vitest setupFiles 自动加载）
└── utils.tsx             ← 带 ThemeProvider 的 customRender，所有测试文件统一从此导入 render
```

所有组件统一从 `src/index.ts` 汇总导出。

## 十一、常用代码示例

### 快速使用 ThemeProvider

```tsx
// main.tsx
import { ThemeProvider } from '@anni-ui/react'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <ThemeProvider defaultMode="system">
      <App />
    </ThemeProvider>
  </StrictMode>
)
```

### 主题切换按钮

```tsx
import { useTheme } from '@anni-ui/react'
import { Moon, Sun } from 'lucide-react'

function ThemeToggle() {
  const { theme, toggle } = useTheme()
  return (
    <button onClick={toggle}>
      {theme === 'dark' ? <Sun size={16} /> : <Moon size={16} />}
    </button>
  )
}
```

### 典型表单

```tsx
import { useForm } from 'react-hook-form'
import { Form, FormItem, FormLabel, FormControl, FormMessage, Input, Button } from '@anni-ui/react'

type Fields = { username: string; email: string }

export default function LoginForm() {
  const form = useForm<Fields>()
  return (
    <Form {...form} onSubmit={form.handleSubmit(console.log)}>
      <FormItem>
        <FormLabel required>用户名</FormLabel>
        <FormControl>
          <Input placeholder="请输入用户名" {...form.register('username', { required: '不能为空' })} />
        </FormControl>
        <FormMessage />
      </FormItem>
      <Button type="submit">登录</Button>
    </Form>
  )
}
```

### Container 布局（顶栏 + 侧边栏 + 内容区）

```tsx
import { Container, Header, Aside, Main } from '@anni-ui/react'

<Container>
  <Header className="h-15">顶栏</Header>
  <Container>
    <Aside className="w-50">侧边栏</Aside>
    <Main>主内容区</Main>
  </Container>
</Container>
```

### 命令式 Toast

```tsx
import { Toast } from '@anni-ui/react'

Toast.success('保存成功！')
Toast.error('网络异常，请重试', { duration: 5000 })
```

## 十二、测试体系

### 12.1 测试环境

基于 **Vitest + @testing-library/react** 构建，与 Vite 共享同一份 `vite.config.ts`，零额外转译配置。

| 配置项 | 值 |
|--------|----|
| DOM 环境 | jsdom |
| globals | true（无需手动 import describe/it/expect） |
| setupFiles | `src/test/setup.ts`（jest-dom + matchMedia mock） |
| 覆盖率工具 | @vitest/coverage-v8 |
| 覆盖范围 | `src/core/**/*.{ts,tsx}`，排除 `index.ts` 和 `*.types.ts` |

### 12.2 测试命令

| 命令 | 说明 |
|------|------|
| `pnpm test` | 监听模式，文件变更时自动重跑（开发时使用） |
| `pnpm test:run` | 单次运行，不监听（CI 使用） |
| `pnpm test:coverage` | 单次运行并生成覆盖率报告（低于阈值时非零退出） |
