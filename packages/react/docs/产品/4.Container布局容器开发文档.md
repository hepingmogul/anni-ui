## 一、背景与目标

布局容器是中后台系统最基础的页面骨架结构。本组件集的目标是提供一套**语义化、基于 Flexbox 的页面布局容器**，方便快速搭建如"上-中-下"、"左-右"、"侧边栏+内容区"等常见管理后台页面结构。

核心要求：

- 采用 **Flexbox** 布局，自动判断子元素方向（垂直 / 水平）
- 提供 `Container`、`Header`、`Aside`、`Main`、`Footer` 五个语义化子组件
- 支持任意深度**嵌套**组合，满足复杂多区域布局需求
- 纯布局组件，**不承载业务逻辑**，样式最小化，方便上层覆盖
- 全面支持明暗主题（`light` / `dark`）

## 二、组件清单

| 组件名 | 描述 | HTML 语义标签 |
|--------|------|--------------|
| `Container` | 外层容器，管理子元素排列方向 | `<div>` |
| `Header` | 顶栏容器 | `<header>` |
| `Aside` | 侧边栏容器 | `<aside>` |
| `Main` | 主内容区容器 | `<main>` |
| `Footer` | 底栏容器 | `<footer>` |

> **注意**：`Container` 的直接子元素必须是以上五个组件之一（或嵌套的 `Container`）。`Header`、`Aside`、`Main`、`Footer` 的父元素必须是 `Container`。

## 三、使用结构

```tsx
import { Container, Header, Aside, Main, Footer } from '@anni-ui/react';

// 基础：上下布局
<Container>
  <Header>顶栏内容</Header>
  <Main>主内容</Main>
</Container>

// 侧边栏布局（左右）
<Container>
  <Aside>侧边栏</Aside>
  <Main>主内容</Main>
</Container>

// 复杂嵌套
<Container>
  <Header>顶栏</Header>
  <Container>
    <Aside>侧边栏</Aside>
    <Main>主内容</Main>
  </Container>
  <Footer>底栏</Footer>
</Container>
```

## 五、组件详细设计

### 5.1 Container

**职责**：外层容器，通过检测直接子元素中是否存在 `Header` 或 `Footer` 来自动决定 `flex-direction`：
- 存在 `Header` 或 `Footer` → `flex-direction: column`（垂直排列）
- 否则 → `flex-direction: row`（水平排列，默认值）

也可通过 `direction` 属性手动指定排列方向，覆盖自动推导逻辑。

**Props**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `direction` | `'horizontal' \| 'vertical'` | 自动推导 | 子元素排列方向 |
| `className` | `string` | — | 追加 Tailwind 类名 |
| `style` | `CSSProperties` | — | 内联样式 |
| `children` | `ReactNode` | — | 子元素 |

**自动推导规则**

```tsx
// 伪代码逻辑
const hasHeaderOrFooter = React.Children.toArray(children).some(
  (child) =>
    React.isValidElement(child) &&
    (child.type === Header || child.type === Footer)
);
const flexDirection = direction ?? (hasHeaderOrFooter ? 'column' : 'row');
```

**DOM 输出**

```html
<!-- 垂直排列（含 Header / Footer） -->
<div class="flex flex-1 basis-auto min-w-0 box-border flex-col">
  <!-- children -->
</div>

<!-- 水平排列（仅含 Aside / Main） -->
<div class="flex flex-1 basis-auto min-w-0 box-border flex-row">
  <!-- children -->
</div>
```

### 5.2 Header

**职责**：顶栏容器，通常放置导航栏、品牌 Logo、用户信息等。高度不设默认值，由业务层通过 `className`（如 `h-15`）或 `style` 自由控制。

**Props**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `className` | `string` | — | 追加 Tailwind 类名，高度推荐用 `h-*` 类控制 |
| `style` | `CSSProperties` | — | 内联样式 |
| `children` | `ReactNode` | — | 子元素 |

**DOM 输出**

```html
<!-- 通过 className 控制高度 -->
<header class="flex items-center shrink-0 box-border px-5 h-15 dark:bg-neutral-800">
  <!-- children -->
</header>
```

## 七、样式规范

样式全部通过 **Tailwind CSS 工具类**实现，不引入额外 CSS 文件。使用项目统一的 `cn()` 工具函数合并类名。

### 7.1 各组件 Tailwind 类名

#### Container

| 场景 | Tailwind 类名 |
|------|--------------|
| 基础容器 | `flex flex-1 basis-auto min-w-0 box-border` |
| 垂直排列（有 Header / Footer） | `flex-col` |
| 水平排列（无 Header / Footer） | `flex-row` |

#### Header

| 场景 | Tailwind 类名 |
|------|--------------|
| 基础顶栏 | `flex items-center shrink-0 box-border px-5` |
| 暗色主题 | `dark:bg-neutral-800` |

> 高度无内置默认值，由业务层通过 `className` 传入 `h-*` 类（如 `h-15`）或通过 `style` 内联设置。

#### Aside

| 场景 | Tailwind 类名 |
|------|--------------|
| 基础侧边栏 | `flex flex-col shrink-0 box-border overflow-auto` |
| 暗色主题 | `dark:bg-neutral-800` |

> 宽度无内置默认值，由业务层通过 `className` 传入 `w-*` 类（如 `w-50`）或通过 `style` 内联设置。

#### Main

| 场景 | Tailwind 类名 |
|------|--------------|
| 基础主内容区 | `flex-1 basis-auto overflow-auto box-border p-5` |

#### Footer

| 场景 | Tailwind 类名 |
|------|--------------|
| 基础底栏 | `flex items-center shrink-0 box-border px-5` |
| 暗色主题 | `dark:bg-neutral-800` |

> 高度无内置默认值，由业务层通过 `className` 传入 `h-*` 类（如 `h-15`）或通过 `style` 内联设置。

### 7.2 组件实现参考

```tsx
// Container.tsx
import { cn } from '../utils/cn'
import React from 'react'
import { Header } from './Header'
import { Footer } from './Footer'

export const Container = ({ direction, className, style, children }: ContainerProps) => {
  const isVertical =
    direction === 'vertical' ||
    (direction === undefined &&
      React.Children.toArray(children).some(
        (child) =>
          React.isValidElement(child) &&
          (child.type === Header || child.type === Footer),
      ))

  return (
    <div
      className={cn(
        'flex flex-1 basis-auto min-w-0 box-border',
        isVertical ? 'flex-col' : 'flex-row',
        className,
      )}
      style={style}
    >
      {children}
    </div>
  )
}
```

### 7.3 暗色主题说明

项目 `tailwind.config.js` 已配置 `darkMode: 'class'`，在根元素添加 `.dark` 类后，`dark:` 前缀工具类自动生效。布局容器本身背景色默认透明，背景色由业务层通过 `className` 传入或直接使用 `dark:bg-neutral-800` 等语义色覆盖。

| 组件 | 推荐暗色背景类 |
|------|--------------|
| Header | `dark:bg-neutral-800` |
| Aside | `dark:bg-neutral-900` |
| Footer | `dark:bg-neutral-800` |
| Main | 无（透明，由页面背景决定） |

## 八、无障碍要求

| 要求 | 说明 |
|------|------|
| 语义化 HTML | `Header` 使用 `<header>`，`Aside` 使用 `<aside>`，`Main` 使用 `<main>`，`Footer` 使用 `<footer>`，天然提供语义化 |
| ARIA landmark | 浏览器会将 `<header>`、`<aside>`、`<main>`、`<footer>` 自动映射为对应 ARIA landmark role（`banner`、`complementary`、`main`、`contentinfo`） |
| 多个 Aside / Header | 若页面存在多个 `<aside>` 或 `<header>`，需为每个元素提供可区分的 `aria-label`，避免屏幕阅读器混淆 |
| 跳过导航 | 建议在 `<main>` 上添加 `id="main-content"`，配合页面顶部的"跳过导航"链接（`<a href="#main-content">`）提升键盘用户体验 |
| 焦点管理 | 布局组件本身不应拦截焦点，`tabIndex` 默认不设置 |

## 九、验收标准

### 功能验收

| 验收项 | 说明 |
|--------|------|
| 自动方向推导 | `Container` 有 `Header` 或 `Footer` 子元素时自动切换为垂直排列 |
| 手动覆盖方向 | `direction="horizontal"` 可覆盖自动判断 |
| `Header` 高度控制 | 通过 `className` 传入 `h-*` 类或 `style` 控制高度，无默认值 |
| `Footer` 高度控制 | 通过 `className` 传入 `h-*` 类或 `style` 控制高度，无默认值 |
| `Aside` 宽度控制 | 通过 `className` 传入 `w-*` 类或 `style` 控制宽度，无默认值 |
| `Main` 填充剩余空间 | `flex-1` 使 `Main` 占满剩余区域 |
| 嵌套组合 | 所有 7 种常见布局均可正确渲染 |
| 类名 / 样式透传 | `className` 和 `style` 正确追加至根元素 |
| 暗色主题 | 在 `.dark` 类下，`dark:` 前缀工具类正确生效 |

### 代码规范

| 验收项 | 要求 |
|--------|------|
| TypeScript 类型完整 | 所有 Props 均有类型定义，无 `any` |
| 导出方式 | 所有组件通过 `packages/react/src/core/Container/index.ts` 统一导出 |
| 无 DOM 警告 | 不传递无效 HTML 属性到 DOM 节点 |
| 单元测试覆盖 | 基本渲染、Props 生效、嵌套布局场景均有对应测试用例 |
