# Button 组件优化

## 背景

当前 Button 组件通过单一 `variant` prop 同时控制按钮的**颜色语义**和**形态**，例如 `primary` 表示"实心主色"、`ghost` 表示"透明无边框"。这种设计在简单场景下足够用，但无法满足"描边主色按钮"、"纯文本危险按钮"等形态与颜色的自由组合需求。此外，圆角样式、圆形图标按钮、块级显示、自定义渲染标签等常见能力也尚未支持。

本次优化将**颜色**与**形态**拆分，并补充其余布局与结构类 prop。

---

## 设计思路

| Prop      | 职责             | 控制内容                             |
| --------- | ---------------- | ------------------------------------ |
| `variant` | 颜色语义         | 文字色、背景色、边框色的色系         |
| `outline` | 形态：描边       | 透明背景 + 色系边框                  |
| `plain`   | 形态：纯文本     | 无背景、无边框                       |
| `round`   | 形状：全圆角     | `border-radius` 为胶囊形             |
| `circle`  | 形状：正圆       | 等宽高 + 完全圆形，适合纯图标按钮    |
| `block`   | 布局：块级       | 宽度撑满父容器（`width: 100%`）      |
| `tag`     | 结构：自定义标签 | 渲染为任意 HTML 标签或组件（如 `a`） |

**规则**
- `variant` 单独使用：**实心填充**按钮（现有行为，完全兼容）
- `variant` + `outline`：**描边**按钮
- `variant` + `plain`：**纯文本**按钮
- `outline` 与 `plain` 互斥，同时传入时 `plain` 优先并在开发模式打印警告
- `circle` 与 `round` 互斥，同时传入时 `circle` 优先
- `circle` 建议搭配纯图标使用（无 `children` 文字）

---

## 现状

| Prop / 组合                 | 效果               | 当前支持 |
| --------------------------- | ------------------ | -------- |
| `variant="primary"`         | 实心主色按钮       | ✅        |
| `variant="secondary"`       | 实心中性色按钮     | ✅        |
| `variant="ghost"`           | 透明无边框按钮     | ✅        |

| `circle`                    | 正圆图标按钮       | ❌        |
| `block`                     | 宽度 100%          | ❌        |
| `tag`                       | 自定义渲染标签     | ❌        |

## 需求

### 1. 新增 `outline` prop（描边形态）

**产品描述**

`outline` 是一个布尔 prop，开启后按钮呈现"透明背景 + 当前 `variant` 色系的边框 + 同色系文字"的描边形态。视觉权重低于实心按钮，常用于与主按钮并列的次要操作。

**视觉规格**

| 状态     | 表现                                               |
| -------- | -------------------------------------------------- |
| 默认     | 透明背景 + `variant` 色系边框 + `variant` 色系文字 |
| Hover    | `variant` 色系浅底色（如 `bg-primary/10`）         |
| Active   | `variant` 色系浅底色加深（如 `bg-primary/20`）     |
| Disabled | 边框变灰、文字变灰、鼠标禁用                       |
| Focus    | 与其他形态一致的 focus-ring 样式                   |

**使用场景示例**

- 弹窗底部的"取消"按钮（搭配 `variant="primary"` 的"确认"按钮）
- 筛选器中的"重置"按钮
- 需要强调轮廓但不占视觉重心的次要操作

**示例代码**

```tsx
<Button variant="primary" outline>取消</Button>
<Button variant="danger" outline>删除</Button>
<Button variant="secondary" outline disabled>不可操作</Button>
```

---

### 2. 新增 `plain` prop（纯文本形态）

**产品描述**

`plain` 是一个布尔 prop，开启后按钮无背景、无边框，仅通过 `variant` 色系的文字颜色传达可交互性，点击区域仍完整保留。适合内容密集场景，减少视觉干扰。

**视觉规格**

| 状态     | 表现                                          |
| -------- | --------------------------------------------- |
| 默认     | 无背景、无边框，`variant` 色系文字            |
| Hover    | 文字颜色加深（如 `text-primary-hover`）       |
| Active   | 文字颜色再加深（如 `text-primary-active`）    |
| Disabled | 文字变灰（`text-neutral-400`）、鼠标禁用      |
| Focus    | 与其他形态一致的 focus-ring 样式              |

> 纯文本按钮不展示背景色变化，通过文字颜色变化和光标反馈来传递交互状态。

**使用场景示例**

- 表格操作列中的"详情"、"编辑"等内联操作
- 卡片中的"查看更多"展开入口
- 表单说明旁的"了解更多"跳转链接

**示例代码**

```tsx
<Button variant="primary" plain>查看详情</Button>
<Button variant="danger" plain>删除</Button>
<Button variant="primary" plain icon={ExternalLink} iconPosition="right">了解更多</Button>
```

---

### 3. 新增 `round` prop（胶囊圆角）

**产品描述**

`round` 是一个布尔 prop，开启后按钮的 `border-radius` 变为完整的胶囊形（`rounded-full`），适合强调品牌感或在卡片等柔和场景中使用。

**视觉规格**

- 仅影响圆角半径，颜色、尺寸等其他规格不变
- 与所有 `variant`、`outline`、`plain`、`size` 可自由组合

**示例代码**

```tsx
<Button variant="primary" round>圆角按钮</Button>
<Button variant="primary" outline round>描边圆角</Button>
<Button size="sm" round>Small Round</Button>
```

---

### 4. 新增 `circle` prop（正圆图标按钮）

**产品描述**

`circle` 是一个布尔 prop，开启后按钮强制等宽高（宽度 = 高度 = 当前 `size` 对应高度），并设置为完全圆形，专用于**纯图标**场景。

**视觉规格**

| size  | 宽高  |
| ----- | ----- |
| `sm`  | 28px  |
| `md`  | 36px  |
| `lg`  | 44px  |

- `circle` 开启时，`px` 内边距移除，改为等宽高正方形后圆化
- 建议仅传 `icon`，不传 `children`；若同时传入文字，文字截断不保证美观

**示例代码**

```tsx
<Button variant="primary" circle icon={Search} />
<Button variant="ghost" circle icon={Settings} />
<Button variant="danger" outline circle icon={Trash} />
```

---

### 5. 新增 `block` prop（块级展示）

**产品描述**

`block` 是一个布尔 prop，开启后按钮宽度撑满父容器（`width: 100%`），适合移动端全宽按钮、表单底部操作区、抽屉/卡片底部按钮等场景。

**视觉规格**

- 宽度变为 `100%`，高度和内边距规则不变
- 与所有 `variant`、`outline`、`plain`、`size`、`round` 可自由组合

**示例代码**

```tsx
<Button variant="primary" block>登录</Button>
<Button variant="secondary" outline block>取消</Button>
```

---

### 6. 新增 `tag` prop（自定义渲染标签）

**产品描述**

`tag` 接受一个字符串（HTML 标签名）或 React 组件，用于将按钮渲染为指定元素。默认值为 `'button'`。常见用途：渲染为 `<a>` 标签（外链跳转）或路由组件（如 React Router 的 `Link`）。

**使用场景示例**

- 外链跳转：`tag="a"` + `href` + `target="_blank"`
- 路由导航：`tag={Link}` + `to="/path"`
- 无障碍自定义：`tag="div"` + `role="button"` + `tabIndex={0}`

**示例代码**

```tsx
<Button tag="a" href="https://example.com" target="_blank" variant="primary">
  外链跳转
</Button>

<Button tag={Link} to="/dashboard" variant="secondary" outline>
  进入控制台
</Button>
```

> 使用 `tag` 时，原生 `disabled` 属性在非 `button` 元素上无效，需业务方自行处理禁用逻辑（如添加 `aria-disabled` 并阻止点击）。

## 形态 × 颜色 组合矩阵

|                   | `primary` | `secondary` | `danger` | `ghost` |
| ----------------- | --------- | ----------- | -------- | ------- |
| 默认（实心）      | ✅ 已有   | ✅ 已有     | ✅ 已有  | ✅ 已有 |
| `outline`（描边） | ✅ 新增   | ✅ 新增     | ✅ 新增  | 无意义  |
| `plain`（纯文本） | ✅ 新增   | ✅ 新增     | ✅ 新增  | 无意义  |

> `ghost` 本身已是透明无边框，与 `outline`/`plain` 组合无实际区别，不做处理（开发模式打印警告）。

## API 变更

### `ButtonVariant` 类型不变

```ts
export type ButtonVariant = 'primary' | 'secondary' | 'ghost' | 'danger'
```

### `ButtonProps` 完整变更

**变更前**

```ts
export interface ButtonProps extends ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: ButtonVariant
  size?: ButtonSize
  loading?: boolean
  icon?: LucideIcon
  iconPosition?: 'left' | 'right'
  children?: ReactNode
}
```

**变更后**

```ts
export interface ButtonProps extends ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: ButtonVariant
  size?: ButtonSize
  outline?: boolean              // 新增：描边形态
  plain?: boolean                // 新增：纯文本形态
  round?: boolean                // 新增：胶囊圆角
  circle?: boolean               // 新增：正圆图标按钮
  block?: boolean                // 新增：块级 100% 宽度
  tag?: ElementType              // 新增：自定义渲染标签
  loading?: boolean
  icon?: LucideIcon
  iconPosition?: 'left' | 'right'
  children?: ReactNode
}
```

### 向下兼容说明

- 所有新增 prop 均有默认值（`false` 或 `'button'`），未传入时行为与现在完全一致，**无破坏性变更**
- `outline` + `plain` 同时传入：`plain` 优先，开发模式 `console.warn`
- `circle` + `round` 同时传入：`circle` 优先，开发模式 `console.warn`
- `tag` 为非 `button` 元素时，`disabled` 属性不会被传递，需业务方自行处理

## 验收标准

### 功能

- [ ] `outline`：透明背景 + `variant` 色系边框 + 同色文字
- [ ] `plain`：无背景、无边框、`variant` 色系文字
- [ ] `round`：圆角变为胶囊形，不影响其他样式
- [ ] `circle`：等宽高 + 完全圆形，`px` 内边距移除
- [ ] `block`：宽度撑满父容器（`w-full`）
- [ ] `tag`：渲染为指定 HTML 标签或组件，透传额外 props（如 `href`、`to`）
- [ ] `variant` 颜色语义在 `outline` / `plain` 形态下均正确传递
- [ ] 所有新 prop 与 `icon`、`iconPosition`、`loading`、`size` 正常组合
- [ ] `outline` + `plain` 同时传入：`plain` 优先 + 开发警告
- [ ] `circle` + `round` 同时传入：`circle` 优先 + 开发警告
- [ ] 现有 4 种 `variant` 的默认形态不受影响（无回归）
