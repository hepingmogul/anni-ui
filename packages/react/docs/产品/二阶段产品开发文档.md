## 一、阶段目标

### 1.1 二阶段核心目标

在一阶段 11 个基础组件的基础上，完成以下目标：

- **明暗主题系统**：支持 `light` / `dark` / `system` 三种模式，CSS 自定义属性驱动，零额外 JS 运行时开销
- **用户偏好持久化**：主题选择自动写入 `localStorage`，刷新页面后恢复
- **跟随系统**：`system` 模式下实时响应操作系统主题变化（`prefers-color-scheme`）
- **组件库 API 完整导出**：`ThemeProvider`、`useTheme` 对外可用

### 1.2 范围说明

| 事项 | 是否在本阶段 |
|------|:----------:|
| 明暗主题切换（ThemeProvider / useTheme） | ✅ |
| 所有一阶段组件适配暗色变量 | ✅ |
| 主题变量体系重构为 CSS 自定义属性（dark 覆盖） | ✅ |
| Select / Modal / Table 等新组件 | ❌（下一阶段） |
| Storybook 文档站 | ❌（下一阶段） |

## 二、明暗主题系统

### 2.1 设计思路

一阶段的主题系统使用 `:root` 中的 CSS 自定义属性作为颜色令牌，所有组件颜色均引用这些变量，**不硬编码任何颜色值**。

二阶段在此基础上新增 `.dark` 类覆盖策略：

```
:root { --color-neutral-50: #f9fafb; ... }   ← 明亮主题（默认）
.dark { --color-neutral-50: #18181b; ... }   ← 暗色覆盖
```

Tailwind CSS 的 `darkMode: 'class'` 配置使 `dark:` 前缀工具类生效，组件同时使用语义变量与 Tailwind dark 工具类来适配两套主题。

### 2.2 模式说明

| 模式 | 行为 |
|------|------|
| `light` | 始终使用明亮主题，写入 `localStorage` |
| `dark` | 始终使用暗黑主题，写入 `localStorage` |
| `system` | 跟随操作系统 `prefers-color-scheme`，实时响应变化 |

### 2.3 工作原理

```
用户调用 setMode('dark')
    ↓
ThemeProvider 更新 state，写入 localStorage
    ↓
useEffect 执行 document.documentElement.classList.add('dark')
    ↓
:root 中的 CSS 变量被 .dark 规则覆盖
    ↓
所有引用 var(--color-*) 的组件自动更新颜色
```

system 模式额外监听 `MediaQueryList` 的 `change` 事件，操作系统切换主题时无需用户操作即可实时同步。

## 四、API 文档

### 4.1 ThemeProvider

应用级主题上下文提供者，需包裹在应用根节点。

**Props**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `children` | `ReactNode` | — | **是** | 子组件树 |
| `defaultMode` | `ThemeMode` | `'system'` | 否 | 首次加载时的默认模式（若 localStorage 中已有记录则优先读取） |
| `storageKey` | `string` | `'react-theme'` | 否 | localStorage 存储键名 |

**使用示例**

```tsx
// main.tsx
import { ThemeProvider } from '@anni-ui/react'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <BrowserRouter>
      <ThemeProvider defaultMode="system">
        <App />
      </ThemeProvider>
    </BrowserRouter>
  </StrictMode>
)
```

---

### 4.2 useTheme

消费主题上下文的 React Hook，必须在 `<ThemeProvider>` 子树内使用。

**返回值**

| 字段 | 类型 | 说明 |
|------|------|------|
| `theme` | `'light' \| 'dark'` | 当前实际渲染的主题（system 模式已解析为真实值） |
| `mode` | `ThemeMode` | 用户设置的模式（含 `'system'`） |
| `setMode` | `(mode: ThemeMode) => void` | 切换到指定模式 |
| `toggle` | `() => void` | 在 `light` 与 `dark` 之间快速切换 |

**使用示例**

```tsx
import { useTheme } from '@anni-ui/react'
import { Moon, Sun } from 'lucide-react'

function ThemeToggleButton() {
  const { theme, toggle } = useTheme()
  return (
    <button onClick={toggle}>
      {theme === 'dark' ? <Sun size={16} /> : <Moon size={16} />}
    </button>
  )
}
```

---

### 4.3 ThemeMode 类型

```ts
type ThemeMode = 'light' | 'dark' | 'system'
```

## 五、使用指南

### 5.1 快速开始

**第一步：在应用根节点包裹 ThemeProvider**

```tsx
// main.tsx
import { ThemeProvider } from '@anni-ui/react'

<ThemeProvider defaultMode="system">
  <App />
</ThemeProvider>
```

**第二步：在任意子组件中使用 useTheme**

```tsx
import { useTheme } from '@anni-ui/react'

function Header() {
  const { theme, toggle, setMode } = useTheme()

  return (
    <header>
      <button onClick={toggle}>
        当前：{theme === 'dark' ? '🌙 暗黑' : '☀️ 明亮'}
      </button>

      {/* 三模式切换 */}
      <select onChange={e => setMode(e.target.value as ThemeMode)}>
        <option value="light">明亮</option>
        <option value="dark">暗黑</option>
        <option value="system">跟随系统</option>
      </select>
    </header>
  )
}
```

### 5.2 tailwind.config.js 配置

暗色主题功能依赖 Tailwind CSS 的 `class` 模式暗色支持：

```js
// tailwind.config.js
export default {
  darkMode: 'class',   // ← 必须添加此配置
  content: ['./index.html', './src/**/*.{js,ts,jsx,tsx}'],
  // ...
}
```

### 5.3 index.css 暗色变量

在 `index.css` 中补充 `.dark` 覆盖块：

```css
.dark {
  --color-primary: #8b5cf6;
  --color-neutral-50: #18181b;
  --color-neutral-900: #fafafa;
  /* ...其余暗色变量 */
}
```

### 5.4 组件适配 dark 工具类（可选增强）

对于背景、文字等使用 neutral 变量的布局组件，同时添加 Tailwind dark 工具类以获得更精确的控制：

```tsx
<div className="bg-neutral-50 dark:bg-neutral-50 text-neutral-900 dark:text-neutral-900">
  {/* 因为 neutral 变量在 .dark 下已经反转，两者写法等价 */}
  {/* 实际上只需引用 CSS 变量类即可，无需额外 dark: 前缀 */}
</div>
```

> **注意**：由于所有颜色均使用 CSS 自定义属性（`var(--color-*)`），`.dark` 规则覆盖变量值后，所有引用这些变量的 Tailwind 类自动生效，**无需为每个组件单独添加 `dark:` 前缀类**。

## 六、实现细节

### 6.1 文件结构

```
src/core/Theme/
├── index.ts              ← 对外导出入口
├── Theme.types.ts        ← 类型定义（ThemeMode、ThemeContextValue、ThemeProviderProps）
└── ThemeProvider.tsx     ← ThemeProvider 组件 + useTheme Hook
```

### 6.2 初始化流程

```
1. ThemeProvider 挂载
2. 从 localStorage 读取上次选择（若无则使用 defaultMode）
3. 若 mode === 'system'，调用 window.matchMedia 获取系统主题
4. 执行 applyTheme()：向 <html> 添加/移除 .dark class
5. 注册 MediaQueryList 监听器（仅 system 模式）
```

### 6.3 持久化策略

- 使用 `localStorage.setItem(storageKey, mode)` 持久化
- 读取时包裹 `try/catch`，兼容 SSR 或 localStorage 被禁用的环境
- `storageKey` 支持自定义，避免多个微前端应用之间的键名冲突

### 6.4 system 模式响应链

```
OS 切换主题
  → MediaQueryList 触发 change 事件
  → handler 调用 applyTheme()
  → <html> class 变化
  → CSS 变量生效
  → 全部组件重绘（无 React re-render，纯 CSS）
```

### 6.5 与一阶段组件的兼容性

一阶段所有组件均使用语义化颜色变量（`bg-neutral-50`、`text-neutral-900` 等），这些 Tailwind 类背后引用的是 `var(--color-neutral-50)` 等 CSS 自定义属性。`.dark` 块覆盖变量值后，**无需修改任何一阶段组件代码**，主题切换即自动生效。

## 七、验收标准

### 7.1 功能验收

- [ ] `ThemeProvider` 正确挂载，无控制台报错
- [ ] `useTheme` 在 Provider 外使用时抛出明确错误信息
- [ ] `setMode('dark')` 后，`<html>` 上出现 `dark` class
- [ ] `setMode('light')` 后，`<html>` 上 `dark` class 被移除
- [ ] `setMode('system')` 后跟随系统主题，手动切换 OS 主题可验证
- [ ] 刷新页面后主题模式从 `localStorage` 正确恢复
- [ ] `toggle()` 在 light 和 dark 之间来回切换
- [ ] 所有一阶段组件在暗黑模式下颜色正确，无硬编码白色背景残留

### 7.2 视觉验收

- [ ] 明亮模式：背景白/浅灰，文字深灰，品牌色 violet-600
- [ ] 暗黑模式：背景深灰 zinc-900，文字近白，品牌色 violet-500
- [ ] 语义色（danger / success / warning / info）在暗色背景下对比度 ≥ 4.5:1
- [ ] 主题切换过渡流畅，无明显闪烁

### 7.3 兼容性验收

- [ ] Chrome / Firefox / Safari 最新版正常运行
- [ ] 禁用 JavaScript 时页面不崩溃（CSS 变量降级）
- [ ] localStorage 不可用时（隐私模式）不报错，降级为 defaultMode
