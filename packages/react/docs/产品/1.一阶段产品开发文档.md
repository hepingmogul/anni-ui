## 一、产品背景

中后台业务系统对 UI 一致性要求高，但现有通用组件库（如 Ant Design）体量较重，定制主题成本高，且样式与业务耦合深。

**React UI** 是为此诞生的轻量级内部组件库，以 **Design Token** 驱动主题、原子化 CSS 管理样式，旨在为中后台项目提供一套统一、可控、易扩展的基础 UI 能力。

---

## 二、产品目标

### 2.1 一阶段目标（当前）

- 完成基础组件集（交互 / 布局 / 数据展示 / 反馈 / 导航）共 **11 个**组件
- 建立统一的主题变量体系，支持一键换肤
- 保证组件 API 类型安全（TypeScript 全覆盖）
- 满足基础无障碍（a11y）要求

### 2.2 长期目标

- 覆盖中后台 80% 的通用 UI 场景
- 提供配套 Storybook 文档站与单元测试
- 支持暗色主题切换

### 3.2 典型使用场景

- 后台管理系统的表单页、列表页、详情页

- 需要精细控制组件 API 的中小规模前端团队

## 四、技术架构

### 4.1 技术选型

| 技术 | 版本要求 | 用途 |
|------|----------|------|
| React | ≥ 18 | 视图层框架 |
| TypeScript | ≥ 5 | 类型安全 |
| Tailwind CSS | ≥ 3 | 原子化样式 + 主题变量 |
| Lucide React | 最新 | 图标系统 |
| Vite | ≥ 5 | 开发与构建工具 |

### 4.2 架构设计原则

- **组件独立**：每个组件拥有独立目录（`组件名/index.ts`、`组件名.tsx`、`组件名.types.ts`），互不耦合
- **类型优先**：Props 接口在 `.types.ts` 中单独维护，便于外部消费
- **样式隔离**：所有样式通过 Tailwind 工具类表达，不引入 CSS Modules 或 styled-components
- **主题集中**：颜色、圆角、阴影等设计决策全部收归 `tailwind.config.js` 的主题扩展，组件不硬编码颜色值

### 5.2 交互类组件

#### Button — 按钮

**功能描述**

核心操作触发组件，支持多种语义变体、三种尺寸、加载状态及图标位置控制。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `variant` | `primary \| secondary \| ghost \| danger` | `primary` | 否 | 语义变体，决定颜色风格 |
| `size` | `sm \| md \| lg` | `md` | 否 | 尺寸规格 |
| `loading` | `boolean` | `false` | 否 | 加载态，自动展示旋转图标并禁用交互 |
| `icon` | `LucideIcon` | — | 否 | 图标组件（来自 lucide-react） |
| `iconPosition` | `left \| right` | `left` | 否 | 图标相对文字的位置 |
| `disabled` | `boolean` | — | 否 | 禁用状态 |

**交互规则**

- `loading` 为 `true` 时，`disabled` 属性自动生效，按钮不可点击
- 图标尺寸随 `size` 自动缩放：`sm` → 12px，`md` → 14px，`lg` → 16px
- 焦点环仅在键盘导航时显示（`focus-visible`），鼠标点击不触发

**使用示例**

```tsx
<Button variant="primary" size="md" icon={Search} loading={false}>
  搜索
</Button>
<Button variant="danger" disabled>删除</Button>
```

#### Input — 输入框

**功能描述**

表单文本输入组件，支持前缀/后缀插槽、标签绑定、错误状态与错误提示。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `size` | `sm \| md \| lg` | `md` | 否 | 尺寸规格 |
| `label` | `string` | — | 否 | 标签文本，自动生成 `id` 并绑定 `<label for>` |
| `prefix` | `ReactNode` | — | 否 | 前缀插槽，渲染于输入框左侧 |
| `suffix` | `ReactNode` | — | 否 | 后缀插槽，渲染于输入框右侧 |
| `error` | `boolean` | `false` | 否 | 错误状态，边框与焦点环切换为 danger 色 |
| `errorMessage` | `string` | — | 否 | 错误提示文本，仅 `error=true` 时显示 |
| `disabled` | `boolean` | — | 否 | 禁用状态，背景变灰，交互锁定 |

**交互规则**

- `label` 未传 `id` 时，自动以 `label` 文本转小写+连字符生成 `id`
- `prefix` / `suffix` 存在时，对应侧的圆角自动移除，保持视觉整体性
- 错误状态的 `errorMessage` 依赖 `error=true` 才会渲染，两者需同时传入

**使用示例**

```tsx
<Input
  label="邮箱"
  placeholder="example@mail.com"
  prefix={<MailIcon size={14} />}
  error
  errorMessage="请输入有效的邮箱地址"
/>
```

#### Checkbox — 复选框

**功能描述**

支持三态（未选 / 已选 / 半选）的复选框，可用于全选场景中的父级节点。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `label` | `ReactNode` | — | 否 | 复选框标签内容 |
| `indeterminate` | `boolean` | `false` | 否 | 半选状态（介于全选与未选之间） |
| `error` | `boolean` | `false` | 否 | 错误状态，标签文字变为 danger 色 |
| `disabled` | `boolean` | — | 否 | 禁用状态 |

**交互规则**

- `indeterminate` 通过 `useRef` 直接设置原生 `input.indeterminate` 属性，与浏览器行为完全一致
- 组件继承全量 `InputHTMLAttributes<HTMLInputElement>`，`onChange`、`checked`、`value` 等均可直接传入

**使用示例**

```tsx
<Checkbox label="全选" indeterminate={someSelected && !allSelected} checked={allSelected} onChange={handleSelectAll} />
<Checkbox label="选项 A" checked={checkedA} onChange={() => setCheckedA(v => !v)} />
```

### 5.3 布局类组件

#### Space — 间距容器

**功能描述**

基于 Flexbox 的间距布局容器，快速控制子元素的排列方向、间距与对齐。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `direction` | `horizontal \| vertical` | `horizontal` | 否 | 排列方向 |
| `size` | `xs \| sm \| md \| lg \| xl \| number` | `md` | 否 | 间距（预设值或像素数值） |
| `wrap` | `boolean` | `false` | 否 | 子元素超出时是否换行 |
| `align` | `start \| center \| end \| baseline` | `center` | 否 | 交叉轴对齐方式 |

**间距预设对应关系**

| 预设值 | Tailwind gap 类 | 像素值（参考） |
|--------|-----------------|--------------|
| `xs` | `gap-1` | 4px |
| `sm` | `gap-2` | 8px |
| `md` | `gap-4` | 16px |
| `lg` | `gap-6` | 24px |
| `xl` | `gap-8` | 32px |

**使用示例**

```tsx
<Space direction="horizontal" size="sm" wrap>
  <Button>A</Button>
  <Button>B</Button>
  <Button>C</Button>
</Space>
```

### 5.4 数据展示类组件

#### Badge — 徽标数

**功能描述**

在目标元素右上角叠加数字角标或圆点的通知组件，也可作为独立行内徽标使用。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `variant` | `default \| success \| warning \| danger \| info` | `danger` | 否 | 颜色语义 |
| `count` | `number` | — | 否 | 数字值 |
| `max` | `number` | `99` | 否 | 最大显示值，超出后显示 `max+` |
| `dot` | `boolean` | `false` | 否 | 圆点模式，不显示数字 |
| `showZero` | `boolean` | `false` | 否 | `count` 为 0 时是否渲染徽标 |
| `children` | `ReactNode` | — | 否 | 被包裹的目标元素 |

**渲染模式**

- **包裹模式**（有 `children`）：徽标绝对定位于目标右上角（`absolute -top-1 -right-1`）
- **独立模式**（无 `children`）：渲染为行内 `<span>` 徽标

**使用示例**

```tsx
{/* 包裹模式 */}
<Badge count={5}>
  <Button icon={Bell}>通知</Button>
</Badge>

{/* 独立模式 */}
<Badge variant="success" count={3} />
```

#### Card — 卡片

**功能描述**

通用内容容器，提供标题区、内容区、底部区三段式结构，适合信息聚合展示。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `title` | `ReactNode` | — | 否 | 标题；传字符串时自动渲染为 `<h3>`，否则渲染自定义节点 |
| `footer` | `ReactNode` | — | 否 | 底部区内容 |
| `bordered` | `boolean` | `true` | 否 | 是否显示边框 |
| `hoverable` | `boolean` | `false` | 否 | 悬停时是否展示加深阴影并显示指针手型 |
| `bodyClassName` | `string` | — | 否 | 内容区的附加类名 |
| `onClick` | `MouseEventHandler` | — | 否 | 点击事件（常与 `hoverable` 搭配） |

**使用示例**

```tsx
<Card
  title="用户信息"
  footer={<span>最后更新：2026-02-28</span>}
  hoverable
  onClick={() => navigate('/user/1')}
>
  <p>这里是卡片内容。</p>
</Card>
```

#### Tabs — 标签页

**功能描述**

横向选项卡导航组件，支持受控与非受控两种数据模式，选项可单独禁用。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `items` | `TabItem[]` | — | **是** | 选项卡配置数组 |
| `activeKey` | `string` | — | 否 | 受控模式下当前激活的 key |
| `defaultActiveKey` | `string` | — | 否 | 非受控模式下默认激活的 key |
| `onChange` | `(key: string) => void` | — | 否 | 切换时的回调，两种模式均触发 |

**TabItem 结构**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `key` | `string` | **是** | 唯一标识 |
| `label` | `ReactNode` | **是** | 选项卡标题 |
| `children` | `ReactNode` | 否 | 面板内容 |
| `disabled` | `boolean` | 否 | 是否禁用此选项 |

**交互规则**

- `activeKey` 传入时进入受控模式，由父组件完全控制激活状态
- `activeKey` 未传时，组件内部维护状态（非受控），`defaultActiveKey` 决定初始选中项
- 未指定 `defaultActiveKey` 时，默认激活第一项

**使用示例**

```tsx
<Tabs
  defaultActiveKey="overview"
  onChange={(key) => console.log('切换到：', key)}
  items={[
    { key: 'overview', label: '概览', children: <OverviewPanel /> },
    { key: 'detail',   label: '详情', children: <DetailPanel /> },
    { key: 'logs',     label: '日志', disabled: true },
  ]}
/>
```

### 5.5 反馈类组件

#### Spinner — 加载指示器

**功能描述**

单一职责的旋转加载图标，用于表示异步操作进行中的状态。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `size` | `xs \| sm \| md \| lg \| xl` | `md` | 否 | 尺寸规格 |
| `color` | `default \| white \| primary \| success \| danger` | `primary` | 否 | 颜色语义 |

**尺寸对应像素值**

| 规格 | 像素 | 典型场景 |
|------|------|----------|
| `xs` | 12px | 按钮内嵌、行内加载 |
| `sm` | 16px | 小型操作区 |
| `md` | 20px | 通用默认 |
| `lg` | 28px | 卡片/区块加载 |
| `xl` | 36px | 全页加载遮罩 |

**无障碍**：组件携带 `role="status"` 与 `aria-label="加载中"`，符合 WAI-ARIA 规范。

**使用示例**

```tsx
<Spinner size="lg" color="primary" />
```

#### Toast — 消息提示

**功能描述**

全局命令式消息通知，无需在组件树中注册，首次调用时自动挂载 DOM 容器。

**调用 API**

```ts
Toast.success(message, options?)  // 成功提示（绿色）
Toast.error(message, options?)    // 错误提示（红色）
Toast.warning(message, options?)  // 警告提示（黄色）
Toast.info(message, options?)     // 信息提示（蓝色）
Toast.show(message, options?)     // 默认 info 类型
```

**参数说明**

| 参数 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `message` | `string` | — | **是** | 提示内容文本 |
| `options.duration` | `number` | `3000` | 否 | 自动消失时长（毫秒），设为较大值可保持常驻 |

**行为说明**

- 容器节点 `#react-toast-root` 首次调用时懒创建并挂载至 `document.body`，无需在应用入口预注册
- Toast 在右上角垂直堆叠，新消息追加到末尾
- 超时后自动移除；用户也可点击右侧关闭按钮提前消除
- 入场动画：`fade-in` + `slide-in-from-right`

**使用示例**

```tsx
<Button onClick={() => Toast.success('保存成功！')}>保存</Button>
<Button onClick={() => Toast.error('网络异常，请重试', { duration: 5000 })}>提交</Button>
```

### 5.6 导航类组件

#### Breadcrumb — 面包屑

**功能描述**

路径层级导航组件，自动在条目间插入分隔符，末级节点高亮标记为当前页。

**Props 定义**

| 属性 | 类型 | 默认值 | 必填 | 说明 |
|------|------|--------|:----:|------|
| `items` | `BreadcrumbItem[]` | — | **是** | 导航条目配置数组 |
| `separator` | `ReactNode` | `<ChevronRight />` | 否 | 自定义分隔符 |

**BreadcrumbItem 结构**

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `label` | `ReactNode` | **是** | 条目显示内容 |
| `href` | `string` | 否 | 存在时渲染为 `<a>` 标签跳转链接 |
| `onClick` | `(e: MouseEvent) => void` | 否 | 点击回调（`href` 与 `onClick` 可同时存在） |

**渲染规则**

- 最后一项（当前页）渲染为 `<span>`，携带 `aria-current="page"`，不可交互
- 非末项：`href` 存在时渲染 `<a>`，否则渲染 `<button>`
- 整体由 `<nav aria-label="breadcrumb">` 包裹，符合语义化要求

**使用示例**

```tsx
<Breadcrumb
  items={[
    { label: '首页', href: '/' },
    { label: '用户管理', href: '/users' },
    { label: '用户详情' },
  ]}
/>
```

## 六、设计规范

### 6.1 语义化颜色变量

所有组件颜色均引用以下语义变量，统一在 `tailwind.config.js` 中定义，不在组件内硬编码颜色值。

| 变量前缀 | 语义 | 衍生变量 |
|----------|------|----------|
| `primary` | 品牌主色 | `primary-hover`、`primary-active`、`primary-disabled`、`primary-ring` |
| `danger` | 危险 / 错误 | `danger-hover`、`danger-active`、`danger-disabled`、`danger-light`、`danger-fg` |
| `success` | 成功 | `success-light`、`success-fg` |
| `warning` | 警告 | `warning-light`、`warning-fg` |
| `info` | 信息 | `info-light`、`info-fg` |
| `neutral` | 中性灰阶 | `neutral-50` ～ `neutral-900` |

### 6.2 统一尺寸规范

支持 `size` 属性的组件（`Button`、`Input`、`Spinner` 等）统一采用以下三档标准：

| 规格 | 高度 | 字号 | 水平内边距 | 典型场景 |
|------|------|------|------------|----------|
| `sm` | 28px | 12px（xs） | 12px | 工具栏、密集列表 |
| `md` | 36px | 14px（sm） | 16px | 通用表单、默认场景 |
| `lg` | 44px | 16px（base） | 20px | 重要操作、移动端适配 |

### 6.3 交互状态规范

所有可交互组件须覆盖以下五种状态：

| 状态 | 视觉表现 |
|------|----------|
| 默认（default） | 基础样式 |
| 悬停（hover） | 背景/颜色微变深 |
| 激活（active/pressed） | 背景/颜色更深，模拟按压 |
| 聚焦（focus-visible） | 显示 2px 焦点环，offset 2px |
| 禁用（disabled） | 降低对比度，`cursor-not-allowed` |
