## 一、背景与目标

表单是中后台系统最高频的交互场景。本阶段目标是在一阶段基础组件之上，构建一套**完整的受控表单体系**，核心要求：

- 集成 **react-hook-form**，统一表单状态管理与验证
- `Form` 提供上下文，`FormItem` / `FormLabel` / `FormControl` / `FormMessage` 负责结构与错误展示，各表单控件专注 UI
- `Input`、`Checkbox` **完全重写**，旧实现废弃，以表单体系为标准重新设计 API
- 支持整体禁用、统一尺寸、标签布局三种表单级配置

## 二、依赖说明

### react-hook-form

表单状态管理核心库，提供 `useForm`、`FormProvider`、`Controller`、`useFormContext`、`useFormState` 等 API。

- 文档：https://www.react-hook-form.com/api/useform/
- 安装：`pnpm add react-hook-form`

**基本用法示例：**

```tsx
import { useForm, SubmitHandler } from "react-hook-form";

type Inputs = {
  username: string;
  age: number;
};

export default function Demo() {
  const { register, handleSubmit, formState: { errors } } = useForm<Inputs>();
  const onSubmit: SubmitHandler<Inputs> = (data) => console.log(data);

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register("username", { required: "用户名不能为空" })} />
      {errors.username && <span>{errors.username.message}</span>}
      <input type="submit" />
    </form>
  );
}
```

## 四、使用结构

```tsx
import { useForm } from "react-hook-form";
import { Form, FormItem, FormLabel, FormControl, FormMessage, FormDescription } from "@anni-ui/react";
import { Input } from "@anni-ui/react";

type FormValues = {
  username: string;
  age: number;
};

export default function Demo() {
  const form = useForm<FormValues>();

  return (
    <Form {...form} onSubmit={form.handleSubmit(console.log)}>
      <FormItem>
        <FormLabel>用户名</FormLabel>
        <FormControl>
          <Input placeholder="请输入用户名" {...form.register("username", { required: "不能为空" })} />
        </FormControl>
        <FormDescription>用于登录的唯一标识</FormDescription>
        <FormMessage />
      </FormItem>

      <FormItem>
        <FormLabel>年龄</FormLabel>
        <FormControl>
          <InputNumber min={1} max={120} {...form.register("age")} />
        </FormControl>
        <FormMessage />
      </FormItem>

      <button type="submit">提交</button>
    </Form>
  );
}
```

**内联布局：**

```tsx
<Form inline {...form} onSubmit={form.handleSubmit(console.log)}>
  <FormItem>
    <FormLabel>关键词</FormLabel>
    <FormControl><Input /></FormControl>
  </FormItem>
  <FormItem>
    <FormLabel>状态</FormLabel>
    <FormControl><Select options={statusOptions} /></FormControl>
  </FormItem>
  <button type="submit">搜索</button>
</Form>
```

## 五、组件详细设计

### 5.1 Form

表单容器，基于 `react-hook-form` 的 `FormProvider` 封装，额外提供布局与配置的 Context。

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `inline` | `boolean` | `false` | `true` 时表单项横向排列 |
| `labelPosition` | `'left' \| 'right' \| 'top'` | `'top'` | 标签位置（`left`/`right` 需配合 `labelWidth`） |
| `labelWidth` | `string \| 'auto'` | `'auto'` | 标签宽度，如 `'80px'`，子 FormItem 继承 |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | 控制表单内所有控件的尺寸 |
| `disabled` | `boolean` | `false` | 禁用整个表单所有控件 |
| `onSubmit` | `FormEventHandler` | — | 表单提交事件，推荐传入 `handleSubmit(onValid)` |
| `children` | `ReactNode` | — | — |

> `Form` 同时透传 `useForm` 返回的所有方法（通过 `FormProvider`），子组件通过 `useFormContext()` 访问。

---

### 5.2 FormItem

单个表单项的容器，内部通过 `React.useId()` 生成唯一 id，供 `FormLabel`、`FormControl`、`FormMessage` 共享。

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `className` | `string` | — | 自定义样式 |
| `children` | `ReactNode` | — | 通常包含 FormLabel + FormControl + FormMessage |

> 继承 `<div>` 的所有原生属性。

---

### 5.3 FormLabel

与 `FormItem` 的 id 自动绑定（`htmlFor`），当字段有验证错误时自动变为错误色。

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `required` | `boolean` | `false` | 显示必填星号 `*` |
| `className` | `string` | — | 自定义样式 |
| `children` | `ReactNode` | — | 标签文本 |

---

### 5.4 FormControl

通过 `Slot` 模式将 `id`、`aria-describedby`、`aria-invalid` 注入子控件，实现 a11y 关联。

**Props：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `children` | `ReactNode` | 表单控件（Input / Select 等），必须为单一根节点 |

> 需配合 `FormField`（react-hook-form Controller）使用，才能自动读取 error 状态。

---

### 5.5 FormMessage

自动读取当前字段的 `error.message`，有错误时渲染红色提示文本，无错误时渲染 `children`（可作为静态提示）。

**Props：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `className` | `string` | 自定义样式 |
| `children` | `ReactNode` | 无错误时展示的默认文本（可选） |

---

### 5.6 FormDescription

字段辅助说明文本，灰色小字，位于控件下方。

**Props：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `className` | `string` | 自定义样式 |
| `children` | `ReactNode` | 说明文本 |

---

### 5.7 FormInputWords

输入字数统计，绑定当前 `FormField` 的字段值，超出限制时变红。

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `max` | `number` | — | **必填**，最大字符数 |
| `layout` | `'absolute' \| 'static'` | `'absolute'` | `absolute` 定位在输入框右下角；`static` 跟随文档流 |
| `className` | `string` | — | 自定义样式 |

> 须在 `FormField` 内部使用，通过 `useFormField` 获取字段名，再通过 `watch` 获取实时值。

### 5.8 Input（重写）

**设计原则：**

- 组件只负责输入框 UI，**不内置 label、error 提示、errorMessage**，这些全部由 `FormLabel` / `FormControl` / `FormMessage` 负责
- 错误状态通过 `FormControl` 注入的 `aria-invalid` 属性驱动样式，Input 自身读取该属性展示错误边框
- `prefix` / `suffix` 仅支持图标或纯文字，不做复杂交互（密码显隐、清空按钮等由 `suffix` 配合父层控制）

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | 输入框高度与字号 |
| `prefix` | `ReactNode` | — | 前置插槽（图标或文字），位于输入区左侧，有边框分隔 |
| `suffix` | `ReactNode` | — | 后置插槽（图标或文字），位于输入区右侧，有边框分隔 |
| `disabled` | `boolean` | `false` | 禁用，影响样式与交互 |
| `className` | `string` | — | 透传至内层 `<input>` 元素 |

> 继承 `InputHTMLAttributes<HTMLInputElement>`，除 `size`、`prefix` 外全量透传至原生 `<input>`。

**样式状态：**

| 状态 | 触发条件 | 表现 |
|------|----------|------|
| 默认 | — | 灰色边框 |
| 聚焦 | focus-within | 主题色边框 + ring |
| 错误 | `aria-invalid="true"` | 红色边框 + ring |
| 禁用 | `disabled` | 背景变灰，鼠标禁止，透明度降低 |

**使用示例：**

```tsx
// 独立使用（搜索框等场景）
<Input size="md" prefix={<SearchIcon />} placeholder="搜索..." />

// 配合 Form 体系使用
<FormItem>
  <FormLabel>用户名</FormLabel>
  <FormControl>
    <Input placeholder="请输入用户名" {...register("username")} />
  </FormControl>
  <FormMessage />
</FormItem>
```

### 5.9 Checkbox（重写）

**设计原则：**

- 完全隐藏原生 `<input type="checkbox">`（`sr-only`），保留其在 DOM 中（无障碍 + form 数据提交）
- 用 Tailwind 自定义绘制勾选框外观（边框、勾号、半选横线）
- **不内置 `error` / `label` prop**：错误色通过 `aria-invalid` 驱动，文字标签由 `FormLabel` 负责
- 支持受控与非受控两种模式

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `checked` | `boolean` | — | 受控选中状态 |
| `defaultChecked` | `boolean` | `false` | 非受控初始状态 |
| `indeterminate` | `boolean` | `false` | 半选状态（优先级高于 `checked`） |
| `onChange` | `ChangeEventHandler<HTMLInputElement>` | — | 状态变化回调 |
| `disabled` | `boolean` | `false` | 禁用 |
| `className` | `string` | — | 自定义样式，作用于外层容器 |

**视觉状态：**

| 状态 | 表现 |
|------|------|
| 未选中 | 空白方框，灰色边框 |
| 选中 | 主题色填充 + 白色对勾 |
| 半选（indeterminate） | 主题色填充 + 白色横线 |
| 错误（`aria-invalid`） | 边框变红 |
| 禁用 | 透明度降低，鼠标禁止 |
| 聚焦 | ring 光晕 |

**使用示例：**

```tsx
// 独立使用（带文字需自行在外部加 label）
<label className="flex items-center gap-2">
  <Checkbox checked={checked} onChange={e => setChecked(e.target.checked)} />
  <span>同意用户协议</span>
</label>

// 配合 Form 体系（CheckboxGroup 场景）
<FormItem>
  <FormLabel>权限</FormLabel>
  <FormControl>
    <div className="flex gap-4">
      <label className="flex items-center gap-2">
        <Checkbox {...register("read")} />
        <span>读取</span>
      </label>
      <label className="flex items-center gap-2">
        <Checkbox {...register("write")} />
        <span>写入</span>
      </label>
    </div>
  </FormControl>
  <FormMessage />
</FormItem>
```

### 5.10 InputNumber

数字输入框，在 `Input` 基础上增加步进按钮与数值范围限制。

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `min` | `number` | — | 最小值 |
| `max` | `number` | — | 最大值 |
| `step` | `number` | `1` | 步进值 |
| `precision` | `number` | — | 小数精度（保留几位小数） |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | 尺寸 |
| `disabled` | `boolean` | `false` | 禁用 |
| `placeholder` | `string` | — | 占位提示 |
| `value` | `number` | — | 受控值 |
| `onChange` | `(value: number \| null) => void` | — | 值变化回调 |
| `className` | `string` | — | 自定义样式 |

**交互规则：**
- 点击 `+` / `-` 按钮按 `step` 步进，达到 `min`/`max` 时按钮禁用
- 手动输入非数字字符时不更新值；失焦后若超出范围则自动修正到边界值
- `precision` 不为空时，输入值自动保留指定小数位

## 六、验证规则约定

所有验证规则通过 `react-hook-form` 的 `register` 或 `Controller` 的 `rules` 传入，不在组件内置校验逻辑。常用规则示例：

```tsx
// 必填
{...register("name", { required: "姓名不能为空" })}

// 长度限制
{...register("bio", { maxLength: { value: 200, message: "最多 200 个字符" } })}

// 正则验证
{...register("phone", { pattern: { value: /^1[3-9]\d{9}$/, message: "手机号格式不正确" } })}

// 自定义验证
{...register("age", { validate: v => Number(v) >= 18 || "必须年满 18 岁" })}
```

## 七、无障碍要求

| 组件 | 要求 |
|------|------|
| `FormLabel` | 通过 `htmlFor` 与控件关联 |
| `FormControl` | 注入 `aria-invalid`、`aria-describedby` |
| `FormMessage` | 通过 `id` 被 `aria-describedby` 引用 |
| `Input` | 读取 `aria-invalid` 展示错误边框；独立使用时需传 `aria-label` |
| `Checkbox` | 原生 `input` 保留在 DOM（`sr-only`），自定义外观叠加渲染；读取 `aria-invalid` 展示错误色 |
| `Radio` | `role="radio"`，`RadioGroup` 使用 `role="radiogroup"` |
| `Select` | 暂缓，单独开发文档中说明 |
| `Switch` | `role="switch"`，`aria-checked` 反映当前状态 |
| `Slider` | `role="slider"`，`aria-valuenow`/`min`/`max` |
