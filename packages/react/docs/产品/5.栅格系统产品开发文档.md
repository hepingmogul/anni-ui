## 一、背景与目标

### 1.1 背景

anni-ui 现有布局能力（`Container` / `Header` / `Aside` / `Main` / `Footer`）聚焦于页面级框架布局，无法满足**页面内容区域的精细化列式排版**需求。中后台场景中，表单、卡片、统计指标等内容普遍需要多列网格对齐，现有方案需要开发者手写 Tailwind 类名完成，一致性差、复用困难。

### 1.2 目标

引入 **24 列栅格系统**（`Row` + `Col`），对标 Element Plus Layout，在 anni-ui 的 Tailwind CSS + Design Token 体系内，提供声明式、响应式、无障碍的列布局能力。

### 1.3 参考

- [Element Plus Layout 布局](https://element-plus.org/zh-CN/component/layout)

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| **列单位** | 最小单位为 1，满行为 24（与 Element Plus 对齐） |
| **Row** | 行容器，基于 Flex 布局，负责控制间距（gutter）与对齐方式 |
| **Col** | 列容器，通过 `span` 声明宽度（占几列），支持偏移、推拉、响应式断点 |
| **Gutter** | 行列间距，由 Row 统一管理，通过 CSS 变量或 padding 分发给 Col |

---

## 三、组件 API 设计

### 3.1 Row

**功能定位：** 行容器，基于 `display: flex` 排列子 `Col`，控制列间距与水平/垂直对齐。

#### Props

| 属性名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `gutter` | `number \| [number, number]` | `0` | 列间距（px）。传数组时 `[水平间距, 垂直间距]`，支持换行场景下的行间距 |
| `justify` | `'start' \| 'end' \| 'center' \| 'space-between' \| 'space-around' \| 'space-evenly'` | `'start'` | 主轴（水平）对齐方式 |
| `align` | `'top' \| 'middle' \| 'bottom' \| 'stretch'` | `'top'` | 交叉轴（垂直）对齐方式 |
| `wrap` | `boolean` | `true` | 是否允许换行（`flex-wrap`） |
| `tag` | `React.ElementType` | `'div'` | 渲染的 HTML 标签 |
| `className` | `string` | — | 附加类名 |
| `children` | `React.ReactNode` | — | 子元素，应为 `Col` 组件 |

#### Gutter 实现方案

采用 **负 margin + padding** 经典方案，与 Element Plus 实现一致，避免影响外部布局：

```
Row:   margin-left: -(gutter/2)px; margin-right: -(gutter/2)px;
Col:   padding-left: (gutter/2)px;  padding-right: (gutter/2)px;
```

支持 `[水平, 垂直]` 双向 gutter 时，Row 同时设置 `margin-top` / `margin-bottom`，Col 设置 `padding-top` / `padding-bottom`。

---

### 3.2 Col

**功能定位：** 列容器，声明当前列占据的栅格数，并支持偏移、响应式等能力。

#### Props

| 属性名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `span` | `number`（0–24） | `24` | 占据列数。`0` 时该列隐藏（`display: none`） |
| `offset` | `number`（0–24） | `0` | 左侧偏移列数（`margin-left`） |
| `push` | `number`（0–24） | `0` | 向右移动列数（`position: relative` + `left`） |
| `pull` | `number`（0–24） | `0` | 向左移动列数（`position: relative` + `right`） |
| `xs` | `number \| ColResponsiveProps` | — | `< 576px` 断点配置 |
| `sm` | `number \| ColResponsiveProps` | — | `≥ 576px` 断点配置 |
| `md` | `number \| ColResponsiveProps` | — | `≥ 768px` 断点配置 |
| `lg` | `number \| ColResponsiveProps` | — | `≥ 992px` 断点配置 |
| `xl` | `number \| ColResponsiveProps` | — | `≥ 1200px` 断点配置 |
| `xxl` | `number \| ColResponsiveProps` | — | `≥ 1600px` 断点配置 |
| `tag` | `React.ElementType` | `'div'` | 渲染的 HTML 标签 |
| `className` | `string` | — | 附加类名 |
| `children` | `React.ReactNode` | — | 子内容 |

#### ColResponsiveProps（响应式对象格式）

```ts
interface ColResponsiveProps {
  span?: number    // 列数
  offset?: number  // 偏移
  push?: number    // 向右推
  pull?: number    // 向左拉
}
```

响应式属性支持简写（纯数字表示 `span`）和完整对象两种写法：

```tsx
// 简写：xs 下占 24 列，md 下占 12 列
<Col xs={24} md={12} />

// 完整对象：md 下占 12 列，offset 2
<Col md={{ span: 12, offset: 2 }} />
```

## 四、响应式断点规范

与 Tailwind CSS 默认断点对齐（部分调整以匹配中后台常见分辨率）：

| 断点名 | 最小宽度 | Tailwind 对应前缀 | 适用场景 |
|--------|----------|-------------------|----------|
| `xs` | 0px（无前缀） | — | 移动端竖屏 |
| `sm` | 576px | `sm:` | 移动端横屏 / 平板竖屏 |
| `md` | 768px | `md:` | 平板横屏 |
| `lg` | 992px | `lg:` | 笔记本 / 桌面 |
| `xl` | 1200px | `xl:` | 宽屏桌面 |
| `xxl` | 1600px | `2xl:` | 超宽屏 |

> **实现策略：** 断点样式通过运行时计算注入（`window.matchMedia` + `useState`），或预生成 Tailwind 安全类名列表（safelist），两种方案见第六节技术设计。

## 五、使用场景与示例

### 5.1 基础布局

```tsx
import { Row, Col } from '@anni-ui/react'

// 均分两列
<Row>
  <Col span={12}>左半</Col>
  <Col span={12}>右半</Col>
</Row>

// 三列布局
<Row>
  <Col span={8}>第一列</Col>
  <Col span={8}>第二列</Col>
  <Col span={8}>第三列</Col>
</Row>
```

### 5.2 分栏间隔（Gutter）

```tsx
// 水平间距 16px
<Row gutter={16}>
  <Col span={6}><div className="bg-primary/20 h-9 rounded" /></Col>
  <Col span={6}><div className="bg-primary/20 h-9 rounded" /></Col>
  <Col span={6}><div className="bg-primary/20 h-9 rounded" /></Col>
  <Col span={6}><div className="bg-primary/20 h-9 rounded" /></Col>
</Row>

// 水平 16px + 垂直 8px（换行场景）
<Row gutter={[16, 8]}>
  {Array.from({ length: 6 }).map((_, i) => (
    <Col key={i} span={8}><div className="bg-primary/20 h-9 rounded" /></Col>
  ))}
</Row>
```

### 5.3 混合布局

```tsx
<Row gutter={16}>
  <Col span={16}>主内容</Col>
  <Col span={8}>侧边信息</Col>
</Row>
<Row gutter={16} className="mt-4">
  <Col span={8}>统计卡片 A</Col>
  <Col span={8}>统计卡片 B</Col>
  <Col span={4}>指标 C</Col>
  <Col span={4}>指标 D</Col>
</Row>
```

### 5.4 列偏移（Offset）

```tsx
// 左侧偏移 6 列
<Row gutter={16}>
  <Col span={6}>左列</Col>
  <Col span={6} offset={6}>右列（偏移6）</Col>
</Row>
```

### 5.5 对齐方式（Justify / Align）

```tsx
// 居中对齐
<Row justify="center" align="middle">
  <Col span={4}>居中内容</Col>
  <Col span={4}>居中内容</Col>
</Row>

// 两端对齐
<Row justify="space-between">
  <Col span={6}>左</Col>
  <Col span={6}>中</Col>
  <Col span={6}>右</Col>
</Row>
```

### 5.6 响应式布局

```tsx
// xs: 全宽，sm: 半宽，md: 三分之一，lg: 四分之一
<Row gutter={[16, 16]}>
  {cards.map(card => (
    <Col key={card.id} xs={24} sm={12} md={8} lg={6}>
      <Card title={card.title}>{card.content}</Card>
    </Col>
  ))}
</Row>
```

### 5.7 span=0 隐藏列

```tsx
// 在 xs 断点隐藏（span=0 等价于 display:none）
<Col xs={0} sm={6}>
  仅在 sm 及以上显示
</Col>
```

## 六、技术设计

### 6.1 样式实现方案

**方案 A（推荐）：运行时内联样式（CSS-in-JS Lite）**

- `Row` 通过 React Context 向下传递 `gutter` 值
- `Col` 从 Context 消费 gutter，计算 `padding-left` / `padding-right` 注入 `style` 属性
- `span` / `offset` / `push` / `pull` 对应的宽度和位移通过内联 `style` 计算（`width: (span/24)*100%`、`margin-left: (offset/24)*100%`）
- 响应式断点通过 `useBreakpoint` Hook（`window.matchMedia`）监听，返回当前激活的断点，Col 据此选取对应的 Props 值

**优点：** 零 Tailwind safelist 污染，无需 CSS 文件，与现有 Design Token 体系解耦清晰。

**方案 B（备选）：预生成 Tailwind 安全类名**

- 在 `tailwind.config.js` 的 `safelist` 中枚举所有可能的 `w-[n/24]` 类名
- 优点：纯 CSS，无 JS 计算；缺点：safelist 膨胀（24×6 断点 = 144 条宽度类名），维护成本高

**结论：采用方案 A**，与 Element Plus、Ant Design 主流实现一致。

---

### 6.2 目录结构

Row 与 Col 共属同一功能单元，统一放在 `Row` 文件夹下，源码合并在一个文件中维护：

```
src/core/Row/
├── index.ts          ← 对外导出 Row、Col 及类型
├── Row.tsx           ← Row + Col + RowContext + useBreakpoint 全部实现
├── Row.types.ts      ← RowProps、ColProps、ColResponsiveProps 等类型定义
└── Row.test.tsx      ← Row 与 Col 的单元测试（合并在同一测试文件）
```

**合并为单文件的理由：**

- Row 与 Col 是强耦合组件（Col 必须依赖 RowContext 获取 gutter），拆分为两个文件会产生循环引用风险
- `useBreakpoint` 是 Col 的私有 Hook，无需对外暴露，放在同文件内维护成本更低
- 符合 anni-ui 现有组件目录约定（每个功能单元一个文件夹，实现集中在主文件）

---

### 6.3 Row Context 设计

定义在 `Row.tsx` 内部，不对外导出（私有实现细节）：

```ts
// Row.tsx 内部
interface RowContextValue {
  gutter: [number, number]  // [水平, 垂直]
}

const RowContext = createContext<RowContextValue>({ gutter: [0, 0] })
```

---

### 6.4 useBreakpoint Hook

定义在 `Row.tsx` 内部，作为 Col 的私有 Hook：

```ts
// Row.tsx 内部
type Breakpoint = 'xs' | 'sm' | 'md' | 'lg' | 'xl' | 'xxl'

const BREAKPOINTS: Record<Breakpoint, number> = {
  xs:  0,
  sm:  576,
  md:  768,
  lg:  992,
  xl:  1200,
  xxl: 1600,
}

// 返回当前激活的所有断点（从小到大均为 true，直至不满足）
// 例如 1024px 宽时，xs/sm/md/lg 为 true，xl/xxl 为 false
function useBreakpoint(): Record<Breakpoint, boolean>
```

Col 按优先级（从大到小断点）选取最匹配的响应式 Props 值。

---

### 6.5 类型定义（Row.types.ts）

Row 与 Col 的类型统一定义在同一个 `Row.types.ts` 文件中：

```ts
// Row.types.ts

export type RowJustify = 'start' | 'end' | 'center' | 'space-between' | 'space-around' | 'space-evenly'
export type RowAlign   = 'top' | 'middle' | 'bottom' | 'stretch'

export interface RowProps extends React.HTMLAttributes<HTMLElement> {
  gutter?:  number | [number, number]
  justify?: RowJustify
  align?:   RowAlign
  wrap?:    boolean
  tag?:     React.ElementType
}

export interface ColResponsiveProps {
  span?:   number
  offset?: number
  push?:   number
  pull?:   number
}

export interface ColProps extends React.HTMLAttributes<HTMLElement> {
  span?:   number
  offset?: number
  push?:   number
  pull?:   number
  xs?:     number | ColResponsiveProps
  sm?:     number | ColResponsiveProps
  md?:     number | ColResponsiveProps
  lg?:     number | ColResponsiveProps
  xl?:     number | ColResponsiveProps
  xxl?:    number | ColResponsiveProps
  tag?:    React.ElementType
}
```

## 七、无障碍（a11y）规范

栅格系统为纯布局容器，本身无语义，遵循以下规范：

| 措施 | 说明 |
|------|------|
| `tag` 属性 | 允许语义化覆盖，如 `<Row tag="section">` / `<Col tag="article">` |
| 默认渲染 `<div>` | 不携带多余的 role，避免干扰屏幕阅读器 |
| 隐藏列（span=0）| 使用 `display: none` 而非 `visibility: hidden`，确保 AT 不读取隐藏内容 |
| 不阻断键盘焦点流 | 容器不设 `tabIndex`，焦点顺序由内部子元素决定 |
