## 一、背景与目标

### 1.1 背景

anni-ui 目前缺少统一的文字链接组件。开发者在需要内联文本跳转、操作入口等场景时，只能直接使用原生 `<a>` 标签，导致样式不统一、状态缺失（禁用、hover 等）、无障碍属性不完整等问题。

### 1.2 目标

引入 **Anchor 文字链接组件**，对标 Element Plus Link，在 anni-ui 的 Tailwind CSS + Design Token 体系内，提供语义化、可定制、无障碍的内联链接能力。

### 1.3 参考

- [Element Plus Link 链接](https://element-plus.org/zh-CN/component/link)

---

## 二、核心概念

| 概念 | 说明 |
|------|------|
| **类型（type）** | 链接的语义颜色，对应 Design Token 中的语义色板（default / primary / success / warning / danger / info） |
| **下划线（underline）** | 控制下划线的显示时机：`always`（始终显示）/ `hover`（悬停时显示）/ `never`（从不显示） |
| **禁用（disabled）** | 禁止点击与交互，视觉上灰化，不触发 href 跳转 |
| **图标（icon）** | 链接前置图标，支持传入 SVG React 组件 |
| **asChild** | 将 Anchor 的样式与行为"合并"到唯一子元素上，不额外渲染 `<a>` 标签，用于集成 react-router-dom `<Link>` 等第三方路由组件 |

---

## 三、组件 API 设计

### 3.1 Anchor

**功能定位：** 文字超链接，基于 `<a>` 标签渲染，支持多种语义颜色、下划线控制、禁用态与图标。

#### Props

| 属性名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `type` | `'default' \| 'primary' \| 'success' \| 'warning' \| 'danger' \| 'info'` | `'default'` | 语义类型，控制链接文字颜色 |
| `underline` | `'always' \| 'hover' \| 'never'` | `'hover'` | 下划线显示时机 |
| `disabled` | `boolean` | `false` | 是否禁用，禁用后不可点击，不触发跳转 |
| `href` | `string` | — | 原生 `href` 属性，传入时渲染为可跳转链接 |
| `target` | `'_self' \| '_blank' \| '_parent' \| '_top'` | `'_self'` | 同原生 `target` 属性 |
| `icon` | `React.ComponentType` | — | 前置图标，传入 SVG React 组件 |
| `className` | `string` | — | 附加类名 |
| `asChild` | `boolean` | `false` | 为 `true` 时不渲染 `<a>` 标签，将所有样式类与事件合并（merge）到唯一子元素上 |
| `onClick` | `React.MouseEventHandler<HTMLAnchorElement>` | — | 点击事件，禁用态下不触发 |
| `children` | `React.ReactNode` | — | 链接文本内容。`asChild` 为 `true` 时，子元素必须是单一可接受 `className` / `onClick` 等 HTML 属性的 React 元素 |

#### Slots（对应 React children / render props）

| 插槽名 | 说明 |
|--------|------|
| `children`（default） | 链接的文字内容 |
| `icon` | 前置图标区域，优先级高于 `icon` prop |

## 四、使用场景与示例

### 4.1 基础用法

六种语义类型，覆盖常见链接颜色需求：

```tsx
import { Anchor } from '@anni-ui/react'

<Anchor href="https://example.com" target="_blank">default</Anchor>
<Anchor type="primary">primary</Anchor>
<Anchor type="success">success</Anchor>
<Anchor type="warning">warning</Anchor>
<Anchor type="danger">danger</Anchor>
<Anchor type="info">info</Anchor>
```

### 4.2 禁用状态

```tsx
<Anchor disabled>default</Anchor>
<Anchor type="primary" disabled>primary</Anchor>
<Anchor type="danger" disabled>danger</Anchor>
```

禁用态下：
- 文字颜色使用 `color-disabled`（Design Token）
- 鼠标光标变为 `not-allowed`
- 点击事件不触发，`href` 跳转被阻止

### 4.3 下划线控制

```tsx
// 始终显示下划线
<Anchor underline="always">always</Anchor>

// 悬停时显示（默认）
<Anchor underline="hover">hover</Anchor>

// 从不显示下划线
<Anchor underline="never">never</Anchor>
```

### 4.4 集成 react-router-dom（asChild）

当项目使用 react-router-dom 时，直接用 `href` 会触发全页刷新，无法享受客户端路由。通过 `asChild` 模式，可将 Anchor 的样式与禁用逻辑委托给 react-router-dom 的 `<Link>` 组件，保留客户端导航能力：

```tsx
import { Link } from 'react-router-dom'
import { Anchor } from '@anni-ui/react'

// 普通 href 模式（适用于外部链接或非路由场景）
<Anchor href="https://example.com" target="_blank" type="primary">
  外部链接
</Anchor>

// asChild 模式（适用于应用内路由跳转）
<Anchor asChild type="primary">
  <Link to="/dashboard">控制台</Link>
</Anchor>

// asChild + 禁用
<Anchor asChild disabled type="primary">
  <Link to="/settings">设置</Link>
</Anchor>
```

**行为说明：**

- `asChild` 为 `true` 时，Anchor 不渲染自身的 `<a>` 标签，而是将计算好的 `className`、`aria-disabled`、`onClick`（禁用拦截）等 props 通过 `React.cloneElement` 合并到子元素上
- 子元素必须是**单一 React 元素**，且能接受 `className`、`onClick` 等标准 HTML 属性（react-router-dom `<Link>` 满足此要求）
- `href`、`target` 在 `asChild` 模式下**不生效**，路由由子元素自身的 `to` 等 prop 控制
- `type`、`underline`、`disabled`、`icon` 在两种模式下均正常工作

### 4.5 图标

前置图标通过 `icon` prop 传入 SVG React 组件，在 `asChild` 模式下同样生效：

```tsx
import { Link } from 'react-router-dom'
import { EditIcon } from '@anni-ui/icons'

// 普通模式
<Anchor icon={EditIcon}>编辑</Anchor>

// asChild 模式下图标同样生效
<Anchor asChild icon={EditIcon} type="primary">
  <Link to="/edit">编辑</Link>
</Anchor>
```

## 五、视觉规范

### 5.1 颜色映射（Design Token）

| type | 默认色 | hover 色 | 禁用色 |
|------|--------|----------|--------|
| `default` | `color-text-regular` | `color-primary` | `color-text-disabled` |
| `primary` | `color-primary` | `color-primary-light-3` | `color-text-disabled` |
| `success` | `color-success` | `color-success-light-3` | `color-text-disabled` |
| `warning` | `color-warning` | `color-warning-light-3` | `color-text-disabled` |
| `danger` | `color-danger` | `color-danger-light-3` | `color-text-disabled` |
| `info` | `color-info` | `color-info-light-3` | `color-text-disabled` |

### 5.2 下划线实现

- `always`：`text-decoration: underline`（始终）
- `hover`：默认无下划线，`:hover` 时添加 `text-decoration: underline`
- `never`：始终无下划线

### 5.3 图标对齐

- 图标与文字垂直居中对齐（`align-items: center`，`display: inline-flex`）
- 图标与文字间距：`gap: 4px`（对应 Design Token `spacing-1`）
- 图标默认尺寸：`1em`（与当前字号等高，自适应）

## 六、技术设计

### 6.1 样式实现方案

采用 **Tailwind CSS 工具类 + CSS 变量（Design Token）** 方案：

- 使用 `inline-flex items-center` 实现图标文字对齐
- `type` 颜色通过 Tailwind 工具类映射到 Design Token CSS 变量
- `underline` 控制 `underline` / `no-underline` / `hover:underline` 类名
- `disabled` 状态通过 `pointer-events-none opacity-50` 或专用 disabled 颜色类名实现
- 不使用内联 `style`，保持 Tailwind 类名驱动

### 6.2 asChild 实现方案

`asChild` 模式通过 `React.cloneElement` 将 Anchor 计算好的 props 合并到子元素上，**不引入第三方 Slot 库**：

```ts
// Anchor.tsx 内部核心逻辑（伪代码）
if (asChild) {
  const child = React.Children.only(children) as React.ReactElement
  return React.cloneElement(child, {
    className: cn(anchorClasses, child.props.className),
    onClick: mergeHandlers(handleClick, child.props.onClick),
    'aria-disabled': disabled || undefined,
    tabIndex: disabled ? -1 : child.props.tabIndex,
  })
}
```

**合并规则：**

| prop | 合并策略 |
|------|----------|
| `className` | Anchor 类名在前，子元素类名在后（`cn(anchorClasses, child.props.className)`） |
| `onClick` | 两者均执行；`disabled` 时两者均被拦截，不执行 |
| `aria-disabled` | 由 Anchor 控制，子元素的同名 prop 被覆盖 |
| `tabIndex` | `disabled` 时强制设为 `-1`，否则保留子元素原有值 |
| 其他 props | 子元素原有 props 保留，不被 Anchor 覆盖 |

> `icon` prop 在 `asChild` 模式下，Anchor 仍会在子元素**外层**包裹一个 `inline-flex` 容器来渲染图标，此时渲染结构为：
>
> ```html
> <span class="inline-flex items-center gap-1 [anchor-classes]">
>   <svg aria-hidden="true">...</svg>   <!-- icon -->
>   <a ...>链接文字</a>                  <!-- asChild 子元素 -->
> </span>
> ```

### 6.3 安全性

`href` 属性直接渲染到 `<a>` 标签，需在组件内部对 URL 进行净化，防止 XSS 与开放重定向漏洞：

```ts
const ALLOWED_PROTOCOLS = ['http:', 'https:']

function sanitizeUrl(url: string): string {
  try {
    const parsed = new URL(url, window.location.origin)
    return ALLOWED_PROTOCOLS.includes(parsed.protocol) ? parsed.href : '#'
  } catch {
    return '#'
  }
}
```

组件内部使用 `sanitizeUrl(href)` 替代直接传入的 `href`，未传 `href` 时不渲染 `href` 属性。`asChild` 模式下跳过此净化（路由由子元素自行负责）。

### 6.5 类型定义（Anchor.types.ts）

```ts
export type AnchorType = 'default' | 'primary' | 'success' | 'warning' | 'danger' | 'info'
export type AnchorUnderline = 'always' | 'hover' | 'never'
export type AnchorTarget = '_self' | '_blank' | '_parent' | '_top'

export interface AnchorProps extends Omit<React.AnchorHTMLAttributes<HTMLAnchorElement>, 'href'> {
  type?: AnchorType
  underline?: AnchorUnderline
  disabled?: boolean
  href?: string
  target?: AnchorTarget
  icon?: React.ComponentType<React.SVGProps<SVGSVGElement>>
  /**
   * 为 true 时不渲染 <a> 标签，将样式类与交互 props 合并到唯一子元素上。
   * 适用于集成 react-router-dom <Link> 等第三方路由组件。
   */
  asChild?: boolean
  className?: string
  children?: React.ReactNode
}
```

## 七、无障碍（a11y）规范

| 措施 | 说明 |
|------|------|
| 禁用态 `aria-disabled` | 禁用时添加 `aria-disabled="true"`，向屏幕阅读器声明不可用状态 |
| 禁用态不移除 `href` | 通过 `pointer-events: none` + `tabIndex={-1}` 阻止交互，保留语义 |
| 图标 `aria-hidden` | 纯装饰图标添加 `aria-hidden="true"`，避免屏幕阅读器重复读取 |
| `target="_blank"` 安全 | 自动附加 `rel="noopener noreferrer"`，防止 `window.opener` 攻击 |
| 焦点样式 | `:focus-visible` 状态下显示清晰的轮廓样式，不影响鼠标用户体验 |
